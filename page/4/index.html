Genotyping notes

Enzyme

  • rtaq:
    • < 500bp
    • will attach A to tails
  • pyrobest
    • High resolution, 500-5kbp

Primer

  • 0.05-1μM
  • high concentration of primer may cause non-specific priming or primer-dimers.

template

  • 104 to 107 molecules for a typical PCR
  • For amplification from genomic DNA, 100 to 500ng DNA. measured by NanoDrop.
  • multiplex PCR 2-5 folds amount than a typical one.

Protocols

  • primer extension at 72-74°C for 1 min per kilobase of expected product.

Use PyQuery and Pandoc to grab web content and generate Epub

PyQuery

In the past I used XPath to manipulate HTML tree to grab information from webpages. It simply works, but not very convenient. PyQuery is a python package which support syntax similar to jQuery. I found this package more convenient for filtering and removing nodes.

To install it:

$ pip install pyquery

I wrote a python script in ~/.bin/pyfilter.

  • As you could see, I make my script more adaptive by allowing 2 arguments or 1 argument input. so sometimes, such as online book, this script may grab all information together instead of running script for every single page.
  • Something useful but I didn’t add is PyQuery.filter(), with this one, you could selectively extract text even you cannot distinguish it by class or id. For instance, you may require that all text should be longer than a length and contain or begin with some words.
  • You need to pay more attention on encoding of webpage. Please see the download_text() and write_output() functions.
  • Pandoc use ~ pair to generate subscript. Sometimes it may cause format trouble when converting to ePub. My solution is replace it to Chinese character ～ . They look the same, but without the side effect.

Pandoc

In the official document page, there is a chapter describing how to write an ebook by Pandoc.

I used command below to generate my epub.

$ pandoc -t epub -o book_title.epub --epub-cover-image title_image.jpg book_title.markdown
  • Pandoc will look for epub.css in data-dir, which is normally in ~/.pandoc, if without specific file in --epub-stylesheet by command line.
  • Pandoc generate Table-Of-Contents, but only Header 1 will be include. I used open source freeware Sigil to add header 2 into TOC. It’s easy and free of charge.
  • Pandoc require blank line to separate 2 paragraphs. Strangely, even in Mac OSX line break should be “\n”, but I need to use command below in vim to generate extra line.
    :%s/$/\r/g

Fusion Drive: What does it mean?

Apple just released its most updated iMac 2 days ago which includes an optional component called Fusion Drive. Even though Apple claimed it’s 3 folds faster than normal 7200rpm harddisk for file copying, but is it really worthy to add this component?

Literature Review

  1. wikipedia:

    A Hybrid drive is different from standard hard drives in that it integrates a cache using non-volatile memory or even a small solid-state drive (SSD or SSH drive)
    wikipedia

  2. According to KnowledgeBase article “Mac mini (Late 2012) and iMac (Late 2012): About Fusion Drive” released by Apple:

    Presented as a single volume on your Mac, Fusion Drive automatically and dynamically moves frequently used files to Flash storage for quicker access, while infrequently used items move to the hard disk. As a result you’ll enjoy shorter startup times, and as the system learns how you work you’ll see faster application launches and quicker file access. Fusion Drive manages all this automatically in the background.

    • Do I need to configure Fusion Drive?
      When ordered with a Fusion Drive, your iMac or Mac mini is pre-configured

    • Can external USB, FireWire, or Thunderbolt hard drives be added to Fusion Drive?

    External drives cannot be added to Fusion Drive.

    • The version of Disk Utility that comes with Fusion Drive is unique. Earlier versions of Disk Utility can’t be used with a Fusion Drive.

  3. Detailed introduction by MacObserver:

    • Fusion drive is different. It’s tier model instead of cache model or RAID.

  4. Alternatives:

    • Seagate produces a hybrid drive named Momentus XT Solid State Hybrid Drive. Tom’s hardware, a famous hardware site, had a very detailed benchmark test for it.
    • Nvelo’s Dataplex
    • Intel’s Rapid Storage Technology

Summary

  • You could use space of both the SSD and the mechanic hard disk. It’s a combination, not a cache.
  • The performance, within the limit of available space on SSD, is roughly the same of a SSD.
  • Basically a software implement. Physically, it’s 2 hard disks and the auto-swaping is done by OS.

Could I DIY a Fusion Drive?

Cause it’s an optional component and iMac is still not released until November for 21.5 inch and December for 27 inch. The real cost is still not clear. But in Mac Mini, 1TB fusion drive is 250USD more expensive then standard 1TB 5400rpm harddisk.

Tom’s hardware had a gallery on how to replace harddisk to SSD in iMac.

Because the delay of delivery of the iMac, we still don’t know whether it’s possible to replace hard disk as the gallery showed above. My guess is , at least it will be harder then the old 2011 version iMac. Let us wait and see.

Pandoc on USB and Mac

As blogged before, I choose Pandoc as my dissertation writing tool. On Windows or Mac, there do have Official Binary Setup files. But unluckily, I don’t have administrator privilege to Library iMac. What can I do?

unpkg can extra .pkg file content. I once used it to extra Line.app within official .pkg installer so that even without admin power I could use in on public iMac. But after extraction of pandoc.pkg, I notice it’s installed to /usr/local/ which I don’t have writing access too. even though I can execute the binary under iTerm2. It will complain “Could not find files” cause it searches resource files in /usr/local/share.

I notice the pandoc binary is as large as 27.8M, it must be a static binary and with all haskell platform resources it needs. But I notice a enquiry and it claimed that it’s impossible to build static linked binary on Mac.

showes that only /usr/lib/libz.1.dylib, /usr/lib/libSystem.B.dylib , /usr/lib/libiconv.2.dylib are dynamic linked. So, at least I could get rid of oversize haskell-platform files.

CFBE41o

Kunzelmann et al. in Gruenert group produced this cell line2.

  1. Electrophysiologically, the cells show no cAMP-dependent Cl transport.
  2. However, after treatment with the calcium ionophore, ionomycin, cells secrete Cl, albeit at a lower level than that observed in normal cells.
  3. Genetically, the cells express CFTR mRNA as determined by polymerase chain reaction amplification and CFTR protein as determined by Western hybridization analysis.

TMEM16A in CFBE

Galietta group in 2008 3, performed a global gene expression analysis to identify membrane proteins that are regulated by IL-4, which was thought to upregulate CaCC activity/expression. By siRNA against each of the upregulated proteins, they associated TMEM16A with CaCC activity.

IL-4 (10ng/ml 24h) could upregulate expression of TMEM16A mRNA by 20 folds and activity by 5-6 folds.

Markdown tools for Dissertation: Pandoc

Finally I’m starting my dissertation writing. Since I have relied on Markdown for several months. The simplicity and efficiency really impressed me. So I’m seeking is there a Markdown-based scientific writing environment? My undergraduate thesis is written in LaTeX, but honest speaking, it’s too complicated for formatting. To me, it looks like dark magic,so powerful that you could achieve anything but the learning curve are quite sharp.

According to List of Markdown implementations, firstly, I learnt some concepts:

  • MultiMarkdown MMD is a superset of the Markdown syntax. It adds multiple syntax features (tables, footnotes, and citations, to name a few), in addition to the various output formats listed above (Markdown only creates HTML). Additionally, it builds in “smart” typography for various languages (proper left- and right-sided quotes, for example).

  • Pandoc Pandoc understands a number of useful markdown syntax extensions, including document metadata (title, author, date); footnotes; tables; definition lists; superscript and subscript; strikeout; enhanced ordered lists (start number and numbering style are significant); running example lists; delimited code blocks with syntax highlighting; smart quotes, dashes, and ellipses; markdown inside HTML blocks; and inline LaTeX.

There are several comparison report between these two, like this and this.

Advantages of Pandoc

Supported format

  • Input: markdown and (subsets of) Textile, reStructuredText, HTML, LaTeX, and DocBook XML
  • Output: plain text, markdown, reStructuredText, XHTML, HTML 5, LaTeX (including beamer slide shows), ConTeXt, RTF, DocBook XML, OpenDocument XML, ODT, Word docx, GNU Texinfo, MediaWiki markup, EPUB, Textile, groff man pages, Emacs Org-Mode, AsciiDoc, and Slidy, Slideous, DZSlides, or S5 HTML slide shows. It can also produce PDF output on systems where LaTeX is installed.

Extentions of Markdown

  • Citation
  • Docx output support
  • Template

These 3 are critical for me. So even MMD may be supported by more applications, I prefer pandoc for my dissertation writing. Let’s see what will happen.

Knitr

I first know this is from a chinese blog. When I was undergraduate, I did use R for some of my statistics courses. But mostly my preference was always Python as the primary script language. By embedding R code in markdown document, I could:

  • eliminate the pain of generating figures in Excel and adjust all those formatting stuff.
  • record the real value of my data in the document.
  • use \ to annotate the original datafile which contain data.

Normally I will write my markdown document in R studio.

Citations

I already bought Papers and it could orgainze papers in mannual collections and export them into bib format. So I could use citeproc-hs in pandoc to cite and generate them. Generally speaking, you could either rely

  • on Knitr or
  • Pandoc’s own citation support.

Different csl format could be downloaded from zotero repository. Place the downloaded file into ~./csl then you could use it in pandoc command.

Place your bib database file citations.bib in ~/.pandoc.

Template and working flow

Refering to “Reproducible research with markdown, knitr and pandoc“ and pandoc’s citation options

$ Rscript -e "library(knitr); knit('thesis_example.Rmd')"
$ pandoc --bibliography="citations.bib" --csl="nature" -o thesis_example.docx  -t docx thesis_example.md

Pandoc support reference format docx. According to pandoc references. We could firstly produce our format document by pandoc. Then place it it ~/.pandoc folder, which is the default data directory. Rename it to reference.docx and the format will be applied while the content will be discarded. This setting will overwrite the default format settings,

Finally, create alias in ~/.bash_login (in Mac)

alias meq='pandoc --bibliography="/Users/your_account_name/.pandoc/citations.bib" --csl="nature' -t docx -o QE.docx QE.markdown

pandoc could not understand ~ which is traditionally HOME folder in UNIX/LINUX world, so u need to use the absolute path. Strange.
