Apt and Dpkg Tips

$ apt-cache dump

Lists every package in the cache. This command is especially helpful in
combination with a grep pipe such as

$ apt-cache dump | grep foo

To search for packages whose
names or dependencies
include “foo”.

$ apt-cache policy

Lists the repositories (main/restricted/universe/multiverse) in which a package exists.

$ apt-cache show

Displays information about a binary package.

$ apt-cache showsrc

Displays information about a source package.

$ apt-cache rdepends

Shows reverse dependencies for a package (which packages require the
queried one.

$ dpkg -S

Lists the binary package to which a particular file belongs.

$ dpkg -l

Lists currently installed packages. This is similar to apt-cache dump
but for installed packages.

$ dpkg -c

Lists the contents of a binary package. It is useful for ensuring that
files are installed to the right places.

$ dpkg -f

Shows the control file for a binary package. It is useful for ensuring that
the dependencies are correct.

$ grep-dctrl

Searches for specialized information in packages. It is a specific use of
the grep package (but not installed by default).

$ apt-build

是一个自动优化编译的包，debian体系一直使用i386的体系包，相比gentoo这类源码发行的，速度上确实慢了不少。使用apt-build自动使用新的cpu优化参数编译，这样与gentoo也不差啦。

tex-cjkfonts

在TeX的安装配置中，最麻烦的应该就是字体包的安装了，网上有各种各样的方法，大体上都是由王垠的或者是何勃亮的修改过的gbkfonts版本，制作gbk字体，然后修改map文件，等等等等。但这些方法有些不足：

  1. 不管怎样的用脚本封装好这些复杂的操作，总还是有少许麻烦。
  2. 生成的是GBK的字体，在UTF－8编码环境下不能使用，必须先手动用iconv将编码更改为gbk。
  3. 如果过程中出了什么错，对新手很难知道为什么出错，怎样修改。也就是没有一个好的滚回机制。

而这些缺陷其实只要有一个支持utf8的包管理的机制就可以了，这也是tex-cjkfonts所实现的：制作字体并将之打包成为deb包。如果对这个字体不满意，直接dpkg －r就可以回滚到之前的状态。并且，这个脚本还支持制作utf-8的编码格式的字体，我们可以直接使用，不需要对tex进行转码。

这个包是THU的 oseen 大侠的功绩。在享受方便的同时，多念叨几声～大侠万寿无僵～包的来源是ustc的sid的debian-uo源。经过测试，在sid和feisty下都可以～其实是因为这个本来就只是脚本而已，而且ubuntu的tex没有对debian的进行修改。

要注意的是有两点：

  1. 脚本有两个命令（实际上是一个，另一个是第一个的连接）make-fpkg-sys和make-fpkg。其中前者是制作deb包用的。后者只是建立一个tar.gz，在自己的主目录解压就可以了（这样不需要root权限）。在我的系统上，会有错误，只能生成gz包，所以我自己是把脚本中的判断是不是生成deb和生成gz的部分注释掉了。个人用户可以自己选择。

  2. 生成的字体有个小瑕疵，但我看脚本没看出问题来，所以就没改。具体的说就是：在/usr/share/texmf/tex/latex/CJK/UTF8 目录下生成的c70$facename.fd的内容里，f 都被错误的当作^L了（注意这里的^L是由Ctrl+L键生成的，不是^字符和L字符），所以在使用新utf8字体前，我们要手动修改这个文件。我在vi里就是

    :%s/^L/\f/g

然后就可以使用了。

mysql 5.0的charset 详解

重新配置了一下mysql，新老板的新项目，好好跟师兄混～
恩，闲话休提，正文开始。

本文全部内容以mysql官方网站的document为依据，基本上就是5.0的英文的笔记而已，没什么技术含量。

  1. mysql的charset分为几个级别，server，database，table，column，都可以有不同的charset
  2. mysql的charset定义分为两项，一项是character set（1），是实际的储存的字符形式，还有一项是collation（2），是在查询比对的时候所使用的形式。前者与后者是严格一对多的的关系。每个charset（short for character set）都有一个默认的collation。在定义的时候如果按照这个顺序来确定某一项的charset和collation；

明示 1 和 2 ，则按照 1，2 定义
明示 1 ，则按照 1 和 1 的默认来确定 2
明示 2 ，按照 2 和 2 对应的 1 来定义
均未明示，按照上一级数据单位（如server，database，table）的定义来定义。

注：
collation的命名规范 charset_lan/country_cs大小写敏感/ci大小写不敏感/bin
character set 可以缩写为charset

National Character Set

这是标准sql定义的一个数据类型（NCHAR ｜ NATIONAL CHAR）在mysql5.0里面是默认的utf8。所以下面两行是等价的：

SELECT N'some text';
SELECT _utf8'some text';

当前ucs2不能作为连接的形式，也许下一个版本就可以了吧^_^。

补充一个系统变量的作用character_set_system，这么多种编码，总有一个是内部默认的吧？没错，系统默认的就是由这个变量定义的。默认值是utf8，如果character_set_results没有设置，那么，服务器不知道该怎样返回结果，就会直接用这个内部的形式。

对了，可能有人会问，在utf8的collation中，有这么多种，怎么会这样？utf8不是支持多国语言的吗？是这样的，unicode的支持现在并不是很完整。有两种通用的utf8的collation：

utf8_unicode_ci
utf8_general_ci

其中前者支持扩展的比对，但比对的算法还没能完全实现，而后者则不支持扩展比对，但速度上比较快。对于前者算法不理想的语言种类，会开发一个新的collation。这也就是为什么有这么多种不同的collation的原因。对于我们中文，当然直接用general就可以了。

仕宦当作执金吾

大帅哥刘秀的肺腑之言，一个未来皇帝的人生理想却看来如此很简单。

刘秀，也许不是最伟大的皇帝，但却是我最喜欢的皇帝。喜欢他，也羡慕他，他身上有我理想的人格：平和。

我也不知道应该怎样形容这个温和的皇帝，所以只能选了这个没有什么特点的词。他少年时并没有什么吞并天下的大志，只是老老实实的修自己的身，齐自己的家，但一旦起事，人们却相信这个老实人一定是有理的；他并不是不喜欢辉煌的仪仗，很简单的表达了自己的想法，没有什么掩饰，虚伪，但他的坐骑却长期由家中的牛来充任；他先后迎娶了两位皇后，中间还有地位互换的波折，但他的皇宫却没有惯见的争宠斗争，黑幕血腥，有的只是一个简单的家庭，有点破损，但很快就修补的看不出来。

经常有人说东汉是多么的窝囊，留给我们的映像就是那一拨接一拨的外戚，宦官，还有三国里可怜的献帝——但谁又知道，匈奴是怎样被彻底赶出中华，成为罗马帝国畏惧的上帝之鞭的吗？这是窦太后的族弟，大将军窦宪的功劳。

刘秀平和，用平和的方法教育孩子，所以他的孩子一代一代的要记住母亲的好，但这又有什么错呢？制度在人，后世的不肖让我们忘记了先辈的太平盛世。

不管怎样，这个平和的皇帝在千年之后仍然吸引了我：他的形象不是一个坐在宝座上傲视四方的英雄，他只是简单的微笑，对着他的妻子，对着他的孩子，对着他的子民。胸中有丘壑，但何必尽现于世人呢？

刘秀的目标很简单，生活在这个简单的生活中却是一个不简单的任务，但他做到了。

当他的儿子在知天命的年纪，仍然在睡梦的呼唤中惊醒，因为他见到了父亲母亲，手挽着手，漫步与柳林湖畔的时候，他的老泪又怎能不纵横？
