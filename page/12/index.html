运动会

就要开运动会了，但大四的人了，这种活动自然是不参加的，申请才是硬道理。
但是运动是不能少的，不然怎么吃得开？
晚上跑步，吃了点花生垫底，就撒开了脚丫子，反正抬头望天，自己享受在冷风里飞翔的快乐。

不过出了个搞笑的事，跑到第五圈时，有个大一模样的小师弟气喘吁吁的追上来问：“师兄跑到第几圈了？”感觉有点诡异，但马上反应过来大一的时候参加五千米也是对操场上飞奔的师兄们有畏惧心里的，总觉得怎么自己的体力就这么弱……，于是为了安抚一下freshman，我说“才跑第二圈呢”。结果这小子还是有些不放心，又问了句“师兄是专门练长跑的吗？”我faint，ustc多少年来什么时候招过体育生了？要不是今年羽毛球赛，这个记录应该还能保持，再说，我这样子像吗？有点不耐烦的答了声“不是，就是晚上出来休息一下”。这小子又自顾自的说他自个也是，只不过头天出来运动。呵呵，果然是为校运会出来抱佛脚的，给点年轻人的信心也好，于是再跑了一圈，就装作体力不支，去沙池玩了会，然后重新回来撒脚丫子。

想想自己当年参加的时候，那才真是体育精神啊，根本没想那么多什么名次，反正那也论不到我想，只想着锻炼身体总没有错处的，横竖也是需要。

运动会是一阵风，刮过了之后就没了影，靠这阵风吹起来的动力也就一起没了影。
所以现在也就只有MM能坚持每天五千的任务。MM的心一定是玉做的，温润而坚韧。

我要向MM一样，每天多运动，毕竟，身体是自己的事。

解决unzip乱码问题

在linux下，rar，bz2，gz都好好的，不会出现乱码。就算是有乱码，也可以用convmv很轻松的搞定。但在windows下用winzip压缩的中文文件名和目录不仅是乱码，而且convmv也无法解决。

解决办法：

下载unzip源码包。修改unzpriv.h搜索Convert，然后把下面的宏_OEM_INTERN改为使用_ISO_INTERN，其实，看了前面的注释，大家也就知道为什么会有区别拉！留点空间让大家去尝试啦！

现在convmv可以正常使用了。

利用strace修正错误

最近，电脑升级到了edgy（呵呵，谁叫俺天生是个做小白！），一切都很好，包括与xorg7.1中自带的aiglx，性能经过调节，也已经很好了。但有一个毛病一直没有解决。就是主菜单应用程序这一项点击后总是不停的闪烁，alacarte也变得速度奇慢无比，更本不接受鼠标点击。想要得到他的出错信息又不可得(gnome-panel自动后台，alacarte也不输出)。而且它们也不出错，连报个bug都没资料～

看了一期Debian Planet后受了点启发

$ strace - trace system calls and signals

在New Maintainer Guide中其实也有个小脚本要用到strace的，既然你不报错，有错自己扛，那我就把你的所有行为全部记录下来，嘿嘿，而且alacarte的不响应很象是陷入死循环，看看是哪出错就好了。

果然

$ strace -p $(gnome-panel_id)

之后，发现了反复访问的几个目录中有一个是我没有检查到的

/etc/xdg/menu/debian-menu.menu

嘿嘿，直接去看看，果然这个文件是个死连接，手动删去后，gnome-panel和alacarte的问题都解决了。

加源

如果有dapper的aiglx的源，最前面注释掉。

安装

$ sudo aptitude install libgl1-mesa/edgy libgl1-mesa-dri/edgy cgwd compiz-core compiz-plugins gnome-compiz-manager compiz-gnome cgwd-themes cgwd-themes-extra

将OpenGL的库换成官方的；升级compiz（也是替换掉可能的compiz-vanlilla）；使用新的compiz参数的修改器（当然gconf-editor是最好的，但有点麻烦）；使用新的decorator，使得主题多样化。

$ sudo aptitude remove xserver-xorg-air-core gset-compiz

这两个包在edgy中都不需要了。

配置GDM

修改/etc/gdm/gdm.conf-custom将aiglx服务器一节删去。

调整性能

现在的blur插件会使速度大幅下降。到配置修改器(gconf-editor)中，将blur插件forbid掉。即可。
不过，总的来说，edgy的速度比dapper的要慢。不过插件的数量多了很多。
