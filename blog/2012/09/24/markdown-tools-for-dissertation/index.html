Markdown tools for Dissertation: Pandoc

Finally I’m starting my dissertation writing. Since I have relied on Markdown for several months. The simplicity and efficiency really impressed me. So I’m seeking is there a Markdown-based scientific writing environment? My undergraduate thesis is written in LaTeX, but honest speaking, it’s too complicated for formatting. To me, it looks like dark magic,so powerful that you could achieve anything but the learning curve are quite sharp.

According to List of Markdown implementations, firstly, I learnt some concepts:

  • MultiMarkdown MMD is a superset of the Markdown syntax. It adds multiple syntax features (tables, footnotes, and citations, to name a few), in addition to the various output formats listed above (Markdown only creates HTML). Additionally, it builds in “smart” typography for various languages (proper left- and right-sided quotes, for example).

  • Pandoc Pandoc understands a number of useful markdown syntax extensions, including document metadata (title, author, date); footnotes; tables; definition lists; superscript and subscript; strikeout; enhanced ordered lists (start number and numbering style are significant); running example lists; delimited code blocks with syntax highlighting; smart quotes, dashes, and ellipses; markdown inside HTML blocks; and inline LaTeX.

There are several comparison report between these two, like this and this.

Advantages of Pandoc

Supported format

  • Input: markdown and (subsets of) Textile, reStructuredText, HTML, LaTeX, and DocBook XML
  • Output: plain text, markdown, reStructuredText, XHTML, HTML 5, LaTeX (including beamer slide shows), ConTeXt, RTF, DocBook XML, OpenDocument XML, ODT, Word docx, GNU Texinfo, MediaWiki markup, EPUB, Textile, groff man pages, Emacs Org-Mode, AsciiDoc, and Slidy, Slideous, DZSlides, or S5 HTML slide shows. It can also produce PDF output on systems where LaTeX is installed.

Extentions of Markdown

  • Citation
  • Docx output support
  • Template

These 3 are critical for me. So even MMD may be supported by more applications, I prefer pandoc for my dissertation writing. Let’s see what will happen.

Knitr

I first know this is from a chinese blog. When I was undergraduate, I did use R for some of my statistics courses. But mostly my preference was always Python as the primary script language. By embedding R code in markdown document, I could:

  • eliminate the pain of generating figures in Excel and adjust all those formatting stuff.
  • record the real value of my data in the document.
  • use \ to annotate the original datafile which contain data.

Normally I will write my markdown document in R studio.

Citations

I already bought Papers and it could orgainze papers in mannual collections and export them into bib format. So I could use citeproc-hs in pandoc to cite and generate them. Generally speaking, you could either rely

  • on Knitr or
  • Pandoc’s own citation support.

Different csl format could be downloaded from zotero repository. Place the downloaded file into ~./csl then you could use it in pandoc command.

Place your bib database file citations.bib in ~/.pandoc.

Template and working flow

Refering to “Reproducible research with markdown, knitr and pandoc“ and pandoc’s citation options

$ Rscript -e "library(knitr); knit('thesis_example.Rmd')"
$ pandoc --bibliography="citations.bib" --csl="nature" -o thesis_example.docx  -t docx thesis_example.md

Pandoc support reference format docx. According to pandoc references. We could firstly produce our format document by pandoc. Then place it it ~/.pandoc folder, which is the default data directory. Rename it to reference.docx and the format will be applied while the content will be discarded. This setting will overwrite the default format settings,

Finally, create alias in ~/.bash_login (in Mac)

alias meq='pandoc --bibliography="/Users/your_account_name/.pandoc/citations.bib" --csl="nature' -t docx -o QE.docx QE.markdown

pandoc could not understand ~ which is traditionally HOME folder in UNIX/LINUX world, so u need to use the absolute path. Strange.
